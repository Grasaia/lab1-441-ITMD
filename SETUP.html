üìù Content Customization

1. Update Your Header Information

Open index.html and find the header section (around line 12-37):

<h1 class="name">Your Name</h1>
<p class="title">Aspiring Web Developer</p>

<a href="mailto:your.email@example.com">Email</a>
<a href="https://github.com/yourusername">GitHub</a>
<a href="https://linkedin.com/in/yourprofile">LinkedIn</a>

2. Update Professional Summary

Replace the placeholder text in the summary section with your own (2-3 sentences):

<section class="summary-section">
    <h2 class="section-title">Professional Summary</h2>
    <p class="summary-text">Your summary here...</p>
</section>

3. Update Education

Modify the degree, institution, and graduation date:

<h3 class="degree">Bachelor of Science in [Your Major]</h3>
<p class="institution">Your University Name</p>
<p class="graduation">Expected Graduation: May 2026</p>

4. Update Skills

Edit the skills list. Each skill should be a separate <li> item:

<ul class="skills-list">
    <li class="skill-item">HTML5</li>
    <li class="skill-item">CSS3</li>
    <li class="skill-item">JavaScript</li>
    <!-- Add more skills -->
</ul>

5. Update Work Experience

Add or modify job entries. For each job, update:

<article class="experience-item">
    <h3 class="job-title">Your Job Title</h3>
    <p class="company">Company Name</p>
    <p class="dates">Month Year - Month Year</p>
    <ul class="job-description">
        <li>Responsibility 1</li>
        <li>Responsibility 2</li>
    </ul>
</article>

6. Update Projects

Add or modify project entries with title, description, and links:

<article class="project-item">
    <h3 class="project-title">Project Name</h3>
    <p class="project-description">Brief description...</p>
    <p class="project-tech"><strong>Technologies:</strong> HTML5, CSS3</p>
    <a href="https://github.com/link" class="project-link">View on GitHub</a>
</article>

üñºÔ∏è Adding Your Profile Image

To add a professional headshot or profile image:

  1. Prepare your image:
    • Dimensions: 200x200 pixels (or larger square image)
    • Format: JPG, PNG, or WebP
    • Quality: High resolution, professional-looking
    • File size: Optimized for web (under 200KB recommended)
  2. Save the image:
    • Name it profile.jpg (or update the filename in HTML)
    • Place it in the same folder as index.html
  3. Update the HTML: 
    <img src="profile.jpg" alt="Professional headshot" class="profile-image">

üí° Tip: If you don't have a professional photo yet, you can use a placeholder image from sites like Unsplash or Pexels. Just remember to replace it before publishing!

üé® Styling Customization

Change the Color Scheme

All colors are defined in styles.css. Look for these primary colors:

Element Current Color CSS Property Location
Primary Color (Headers, Links) #0066cc (Blue) color, border-color Throughout CSS
Header Background Linear gradient (Blue) background .header selector
Text Color #333 (Dark Gray) color body selector
Background #f8f9fa (Light Gray) background-color body selector

To change colors: Use find and replace in your editor:

  • Find: #0066cc ‚Üí Replace with your primary color
  • Find: #0052a3 ‚Üí Replace with a darker shade
  • Find: #333 ‚Üí Replace with your text color

Change Fonts

Update the font-family in the body selector of styles.css:

body {
    font-family: 'Your Font Name', fallback-font;
}

Font suggestions:

  • 'Georgia', serif - Classic, professional
  • 'Arial', sans-serif - Clean, modern
  • 'Trebuchet MS', sans-serif - Friendly, modern
  • Google Fonts: https://fonts.google.com

Adjust Spacing

To make the page feel more spacious or compact, modify these values in styles.css:

/* Increase spacing */
margin-bottom: 2rem; /* was 1.5rem */
padding: 3rem 2rem; /* was 2rem 2rem */

/* Decrease spacing */
margin-bottom: 0.5rem; /* was 1rem */
padding: 1rem; /* was 2rem */

üì± Testing Responsiveness

Ensure your site looks good on all devices:

Desktop (1200px+)

  • Full navigation visible
  • Multiple columns for skills
  • Comfortable reading line length

Tablet (768px - 1199px)

  • Adjusted font sizes
  • 2-column skill grid
  • Touch-friendly buttons

Mobile (< 768px)

  • Single column layout
  • Vertical navigation menu
  • Large touch targets

To test:

  1. Open index.html in your browser
  2. Press F12 to open Developer Tools
  3. Click the device toggle (üì± icon) in the top-left
  4. Select different device sizes from the dropdown

üöÄ Deploying to GitHub Pages

Step 1: Prepare Your Files

  • Ensure all files are in your GitHub repository
  • Main file must be named index.html
  • All CSS and images should be in the same directory

Step 2: Enable GitHub Pages

  1. Go to your repository on GitHub
  2. Click Settings tab
  3. Scroll down to GitHub Pages section
  4. Under Source, select main branch
  5. Click Save

Step 3: Access Your Site

After 1-2 minutes, your site will be live at:

https://[yourusername].github.io/[repository-name]

‚úÖ Your resume is now publicly accessible!

üêõ Troubleshooting

Problem: Images Not Showing

Solution: Ensure the image filename matches exactly in the HTML and file system are case-sensitive on some servers.

Problem: Styles Not Applied

Solution: Clear your browser cache (Ctrl+Shift+Delete) and refresh the page.

Problem: GitHub Pages Not Publishing

Solution: Check that your repository is public and GitHub Pages is enabled in Settings.

Problem: Layout Broken on Mobile

Solution: Verify the viewport meta tag is present: <meta name="viewport" content="width=device-width, initial-scale=1.0">
